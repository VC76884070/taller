Actúa como arquitecto de software senior especializado en sistemas ERP y desarrollo full stack.

Estoy desarrollando un SISTEMA DE GESTIÓN INTEGRAL PARA UN TALLER MECÁNICO llamado FURIA MOTOR COMPANY SRL como proyecto de grado universitario.

El sistema debe cumplir una estructura modular por roles y debe ser escalable, seguro y profesional.

────────────────────────────
📌 CONTEXTO GENERAL
────────────────────────────

El sistema controla todo el proceso del taller:

1. Recepción del vehículo
2. Diagnóstico inicial
3. Diagnóstico técnico
4. Cotización
5. Aprobación del cliente
6. Reparación
7. Control de calidad
8. Entrega
9. Control financiero
10. Inventario

El sistema debe ser web y adaptable a móvil.
Debe manejar autenticación y permisos por rol.

────────────────────────────
📌 ROLES DEL SISTEMA (6)
────────────────────────────

1. Jefe Operativo
2. Jefe de Taller
3. Técnico/Mecánico
4. Encargado de Repuestos
5. Administrador General
6. Cliente

Cada rol debe ver solo sus módulos correspondientes.

📌 ESTRUCTURA DETALLADA POR ROL
────────────────────────────

Pestañas:
- Dashboard

- Rendición Diaria
- Comunicados
- Historial
- Perfil
